# IEEE Execution Protocol

## Purpose

This document defines the authoritative CostGuard v17.0 execution profile used for IEEE-grade reproducibility, governance validation, and artifact generation.

## Authoritative Trial Contract

- Trial count: `1`
- Trial seed: `42`
- Domain order:
  1. `D0 = Synthetic`
  2. `L1 = TravisTorrent`
  3. `L2 = BitBrains`
- Runner: `run_ieee_trials.sh`

The repository no longer treats any fourth domain as part of the active publication workflow.

## Dataset Scope

### D0: Synthetic

Generated internally by `SyntheticDataGenerator` using TravisTorrent and BitBrains calibrated distributions.

### L1: TravisTorrent

- Input: `final-2017-01-25.csv`
- Output domain directory: `results/seed_42/real`

### L2: BitBrains

- Input directory: `fastStorage/2013-8`
- Semicolon-separated schema with line-1 header detection
- Output domain directory: `results/seed_42/bitbrains`

## Model and Evaluation Invariants

- `BahdanauBiLSTM`: 3 residual BiLSTM blocks plus Bahdanau attention
- `GATv2Pipeline`: 3 graph-attention layers with dataset-aware feature dimensions
- Validation-only threshold optimisation
- Test evaluation uses the locked validation threshold
- `best_scores.json` top-level keys:
  - `best_lstm`
  - `best_gat`
  - `best_ens`
  - `hpo_triggered`
  - `domain`
  - `seed`

## Lifelong Metric Contract

The unified analytics engine computes T=3 BWT from `bwt_matrix.json` using:

```text
BWT = 0.5 * ((R(D0|theta_L2) - R(D0|theta_D0)) + (R(L1|theta_L2) - R(L1|theta_L1)))
```

Required matrix keys:

- `after_D0`
- `after_L1`
- `after_L2`
